memtest
=======

Just a TextView who's value changes every 200ms, memory grows, app dies.

~~~
devkit release native-android --open
brew install watch
touch ~/Desktop/memory.txt
open ~/Desktop/memory.txt
watch -n5 'adb shell dumpsys meminfo com.example.studio.memtest | grep TOTAL >> ~/Desktop/memory.txt'
~~~

### memtest

Below you can see the output of my test, private dirty memory goes from 20Mb up to 160Mb over 15 minutes,
the test ended when the app force-closed on my phone. On other tests I've seen more a sawtooth pattern in
the memory where (I assume) Garbage Collection runs, however in this test I can't see any evidence of this
happenning here.
